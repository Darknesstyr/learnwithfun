Lesson 1: HTML
BASICS:

1. What is HTML?
HTML (HyperText Markup Language) is the standard language used to create web pages. It structures content on the web using "tags" that define elements such as headings, paragraphs, links, images, and more.


2. Basic HTML Structure
Every HTML document follows a basic structure that includes a declaration of the document type, a root element, a head section for metadata, and a body section for visible content.


3. Common Tags
Headings: Used to create titles and subtitles, ranging from the largest size (Heading 1) to the smallest size (Heading 6).
Paragraphs: Used to add blocks of text.
Links: Used to create clickable text that navigates to other pages or resources.
Images: Used to display pictures on a webpage.
Lists: Used to organize items. They can be unordered (bulleted) or ordered (numbered).


4. Attributes
Attributes provide additional information about an HTML element. For example, a link can include an attribute specifying the URL it points to, or an image can include an attribute describing the image content.


5. Creating Forms
Forms are used to collect user input. They often include text fields, buttons, and other elements to gather and submit information.


6. Semantic HTML
Semantic HTML uses tags that convey meaning and improve accessibility and SEO. For example:
A header section is identified with a "header" tag.
The main content is identified with a "main" tag.
Articles or individual pieces of content are identified with an "article" tag.


7. Inline vs. Block Elements
Inline elements do not start on a new line and only take up as much width as necessary.
Block elements start on a new line and take up the full width available.


8. Comments
Comments are used to add notes within the code. These notes are not visible on the webpage and are used for documentation purposes.


9. Example Project
A basic HTML webpage typically includes a title, a header with a welcome message, a main section with text and images, and a footer with copyright information.


10. Additional Resources
To learn more about HTML, you can explore beginner-friendly tutorials and documentation from trusted sources like W3Schools and MDN Web Docs.



CSS BASICS:

1. What is CSS?
CSS (Cascading Style Sheets) is the language used to style web pages. It controls how HTML elements appear on the screen, such as colors, fonts, layout, spacing, and positioning.



2. Basic Structure of CSS
A CSS rule consists of two main parts:
Selector: Targets the HTML element to apply styles.
Declaration Block: Contains the properties and values to style the selected element.



3. CSS Syntax
CSS follows a straightforward syntax:
Selector: This is the HTML element (e.g., a paragraph, heading) you want to style.
Property: The aspect of the element you want to change (e.g., color, font size).
Value: The setting for the property (e.g., blue, 16px).



4. Types of CSS Selectors
Element Selector: Targets HTML tags (e.g., paragraphs, headings).
Class Selector: Targets elements with a specific class.
ID Selector: Targets elements with a specific ID.
Universal Selector: Targets all elements on the page.



5. CSS Colors
CSS allows you to specify colors in various ways, including:
Named colors: such as red, blue, or green.
Hexadecimal: A six-digit color code (e.g., #FF5733).
RGB: A value based on the Red, Green, and Blue color model (e.g., rgb(255, 87, 51)).
RGBA: RGB with an alpha channel for transparency.



6. CSS Box Model
The CSS box model defines how elements are displayed on a page, consisting of:
Content: The actual content of the element (e.g., text, images).
Padding: Space around the content.
Border: A line surrounding the padding.
Margin: Space outside the border.



7. CSS Layout Techniques
Block vs. Inline Elements: Block elements (like divs) take up the full width, while inline elements (like spans) take up only as much space as needed.
Flexbox: A layout model that enables flexible positioning of elements in a container.
Grid: A system for creating complex layouts using rows and columns.



8. CSS Positioning
CSS offers several types of positioning:
Static: The default positioning for elements, where they flow normally within the document.
Relative: Positioned relative to its normal position.
Absolute: Positioned relative to the nearest positioned ancestor.
Fixed: Positioned relative to the viewport, meaning it stays in place when scrolling.
Sticky: Becomes fixed after scrolling past a certain point.



9. CSS Media Queries
Media queries enable you to apply different styles depending on factors like screen size, orientation, and device type. This is crucial for creating responsive designs.


10. CSS Transitions and Animations
Transitions: Allow smooth changes to CSS properties over time.
Animations: Involve defining keyframes that describe the changes an element goes through during an animation.



11. CSS Best Practices
External Stylesheets: Keeping CSS in separate files helps maintain cleaner, more manageable code.
Organizing CSS: Group styles into logical sections (e.g., layout, typography, colors) for better readability.
Comments: Use comments to explain complex or important parts of the CSS.
Minification: In production environments, use minified CSS to reduce file sizes and improve load times.




JAVASCRIPT BASICS:

1. What is JavaScript?
JavaScript is a dynamic, high-level programming language that is widely used to create interactive effects within web browsers. It allows you to manipulate web page elements, handle user interactions, and perform complex tasks like animations, form validation, and data manipulation.



2. Basic Structure of JavaScript
JavaScript code consists of:
Statements: Instructions that tell the browser what to do (e.g., display a message, perform a calculation).
Expressions: Combinations of variables, values, and operators that produce a result.
Functions: Reusable blocks of code designed to perform a specific task



3. JavaScript Syntax
JavaScript syntax is the set of rules that define a correctly structured JavaScript program. Key elements of JavaScript syntax include:
Variables: Used to store data.
Operators: Symbols used to perform operations on variables and values (e.g., + for addition, - for subtraction).
Control Flow: Statements that control the flow of execution, like loops and conditionals.



4. Variables
Variables are containers for storing data. They can hold different types of information, such as:
String: Textual data enclosed in quotes (e.g., "Hello").
Number: Numeric values (e.g., 25, 3.14).
Boolean: Logical values (true or false).
You declare variables using the let, const, or var keywords, with let and const being preferred in modern JavaScript due to their scoping rules.



5. Data Types
JavaScript has several data types:
Primitive types: These are the most basic types, including numbers, strings, booleans, null, undefined, and symbols.
Objects: More complex data structures used to store collections of values or functions (e.g., arrays, objects).



6. Functions
Functions are blocks of reusable code designed to perform a specific task. They can accept parameters (input values) and return a result. Functions help make your code modular and more manageable.



7. Control Structures
JavaScript provides several control structures for decision-making and repeating actions:
If-else statements: Used for conditional execution.
Switch statements: Used for checking a variable against multiple conditions.
Loops: Allow repeated execution of a block of code:
For loop: Used when you know how many times you want to repeat the action.
While loop: Used when you want to repeat an action as long as a condition is true.
Do-while loop: Similar to a while loop, but the condition is checked after the loop executes.



8. Events
JavaScript enables interaction with HTML elements through events. An event occurs when a user performs an action like clicking a button or moving the mouse. You can use JavaScript to respond to these actions:
Click events: Triggered when an element is clicked.
Mouse events: Triggered by mouse actions like moving or hovering over elements.
Keyboard events: Triggered by key presses.



9. Arrays and Objects
Arrays and objects are fundamental data structures in JavaScript.
Arrays: Ordered collections of data, indexed by numbers.
Objects: Unordered collections of key-value pairs, useful for storing related data.



10. DOM Manipulation
The Document Object Model (DOM) represents the structure of a web page. JavaScript allows you to interact with the DOM by:
Selecting elements: Accessing HTML elements using selectors.
Changing content: Modifying the text or HTML content of elements.
Changing styles: Updating the appearance of elements dynamically.
Adding/removing elements: Creating or deleting elements from the page.



11. Asynchronous JavaScript
JavaScript allows you to run code asynchronously, meaning some tasks can be done while others are running:
Callbacks: Functions passed into other functions that execute after the current task is completed.
Promises: Objects representing the eventual completion (or failure) of an asynchronous operation.
Async/await: Modern syntax for handling asynchronous code more readably.



12. Error Handling
JavaScript provides mechanisms for handling errors in your code:
Try-catch blocks: Used to catch and handle exceptions (errors).
Throw statements: Allows you to create custom errors.



13. JavaScript Best Practices
To write clean, maintainable JavaScript code:
Keep code DRY: Don't Repeat Yourself. Use functions and variables to avoid redundant code.
Write comments: Describe complex logic and functions for clarity.
Follow naming conventions: Use meaningful names for variables and functions.
Modular code: Break down your code into smaller, reusable functions or modules.
